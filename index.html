📞
Patient Calls



What happens
Patient calls from any phone
Call connects to Exotel cloud
Exotel triggers webhook to backend
Session created with caller's number
☁️
Exotel Receives Call



Technical flow
POST /ivr/exotel called with CallSid + From
Backend checks if caller has active token
New caller → language menu
Returning caller → status check
🌐
Language Selection



Patient hears
"Press 1 for Hindi. Press 2 for English."
DTMF input captured by Exotel Gather
Language saved to session in MongoDB
All future prompts in chosen language
🏥
Service Selection



Available services
Press 1 → Doctor queue (D-series tokens)
Press 2 → Lab queue (L-series tokens)
Press 3 → Pharmacy queue (P-series tokens)
Service saved to session
🎫
Token Generated



Token details
Token: D-017 (service prefix + number)
Wait: position × avg service time
Saved to MongoDB with phone + language
Status: "waiting"
📱
Voice + SMS Sent



Patient receives
Voice: "Your token is D-017, wait 20 mins"
SMS: "Token D-017. Expected wait 20 mins"
Hindi version if Hindi was selected
Sent via Exotel SMS API
📊
Dashboard Updates



Admin actions
Token appears in live queue list
Click "Serve" → SMS: "Come to counter"
Click "Done" → SMS: "Service complete"
Filter by service or status

Features

Everything a clinic needs

📞
IVR Call Flow
Fully automated voice menu via Exotel. Patients use keypad digits — no smartphone or internet needed.
🌐
Hindi & English
Complete bilingual support. Voice prompts and SMS messages both adapt to the patient's language choice.
🎫
Smart Token System
Auto-generates D-001, L-001, P-001 style tokens per service with dynamic wait time calculation.
📱
SMS Notifications
3 automated SMS triggers: on token generation, when called to counter, and when service completes.
📊
Live Admin Dashboard
React dashboard with real-time queue view, service/status filters, and one-click token management.
🔁
Automatic Status Check
Returning callers automatically hear their queue position — system identifies them by phone number.

About The Project

💡 The Problem
In India, millions of people visit hospitals and government offices daily, only to spend hours standing in physical queues. There's no way to know your wait time, no way to queue remotely, and no notification when it's your turn. This is especially difficult for elderly patients and people travelling long distances.
Before
Stand in physical queue
No wait time estimate
Miss your turn easily
Need to be present
After
